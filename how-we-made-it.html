The HTML <head> — Setup Before Anything Renders

Line of CodeWhat it does (plain English)
<!DOCTYPE html>Tells the browser: "this file uses modern HTML5 rules, not old 1999 HTML". Without this, browsers go into quirks mode and things look broken.
<html lang="en">Wraps the entire page. lang="en" tells screen readers and search engines the page is in English.
<meta charset="UTF-8">Allows any character to display — English, Devanagari (मराठी), emojis. Without this, special characters show as garbled text like "â€™".
<meta name="viewport" …>Critical for mobile. Tells the phone: "don't zoom out to fit the whole desktop layout on your screen — use my actual screen width." Makes the site mobile-friendly.
<link rel="preconnect" …>Starts a connection to Google's font servers before actually loading fonts. Saves ~200ms because the DNS lookup happens in advance.
<link href="fonts.googleapis…">Loads 4 typefaces from Google Fonts: Playfair Display (headings), Lato (body text), Cormorant Garamond (quotes), Noto Serif Devanagari (मराठी text). All free, streamed from Google's CDN.

The Circular Photo — Three-Layer System

The Problem
Photos are rectangular. A circle crop creates white empty gaps on the sides when the photo is taller than wide. object-fit: cover zooms in and crops the subject. object-fit: contain shows the full subject but leaves gaps.
The Solution — Blur Background Trick
Stack three layers inside a circle that clips everything: (1) a blurred+scaled copy of the same photo fills the gaps with matching colour tones, (2) the sharp photo sits on top at full size. The overflow:hidden circle clips both. The leaf wreath SVG floats outside on a separate layer so it doesn't get clipped.
Layer / CSSWhat it does
.hero-photo-outer { width: 260px }The outermost container. 260px square. The leaf wreath is positioned inside this — not clipped, can extend beyond it.
.hero-photo-wrap { inset: 33px; border-radius: 50%; overflow: hidden }inset: 33px means it sits 33px inside the outer on all sides, making the circle 194px (260−33−33). border-radius: 50% makes it a perfect circle. overflow: hidden clips everything inside to the circle shape.
.hero-blur-bg { width: 260px; filter: blur(40px); transform: scale(1.3) }Same photo, but 260px wide (larger than the 194px circle), centred with translate(-50%,-50%), then blurred 40px and scaled 1.3× so blurred edges never show. Fills circle gaps with matching tones.
.hero-photo { width: 260px; object-fit: contain }The actual sharp photo, also 260px, centred. contain means the whole subject is visible without cropping. Sits above the blur layer via z-index: 2.
.leaf-wreath { top: 19px; left: 19px; width: 222px }The SVG wreath. Positioned inside the outer container (not the clip), 222px wide, offset 19px from the outer edge. This mathematically centres the wreath over the 194px circle. Outside the overflow:hidden so it's not clipped.

Wreath Animation

CSS — wreathSway keyframe animation
@keyframes wreathSway {
  0%   { transform: rotate(-1deg); }
  100% { transform: rotate(1deg);  }
}

/* Applied to .leaf-wreath: */
animation: wreathSway 6s ease-in-out infinite alternate;

alternate means it plays forward then backward, forward then backward — so the wreath sways left, then right, endlessly. ease-in-out means it starts slowly, speeds up in the middle, slows again — like a real leaf in a gentle breeze. 6s is the speed: one full sway takes 6 seconds.

Responsive Font Sizes with clamp()

CSS — clamp() for fluid typography
/* Couple names heading */
font-size: clamp(2rem, 6vw, 3.6rem);
/*         ↑ min   ↑ fluid  ↑ max  */

/* On a 320px phone:  6vw = 19.2px → uses min 2rem (≈32px)  */
/* On a 768px tablet: 6vw = 46px   → uses fluid value       */
/* On a 1440px screen:6vw = 86px   → capped at max 3.6rem   */

This replaces needing 3 different @media breakpoints just for font sizes. One line scales the heading perfectly across every screen width automatically.

IntersectionObserver — Scroll Reveal System

Instead of animating everything on page load (which looks bad), we want each section to animate only when it scrolls into view. The browser has a built-in tool for this called IntersectionObserver. It watches elements and fires a callback when they enter or leave the viewport.

JavaScript — Scroll reveal (lines 1332–1343)
// Step 1: Find all elements that should animate
const revealEls = document.querySelectorAll(
  '.reveal, .reveal-left, .reveal-right'
);

// Step 2: Create an observer with a callback function
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {          // element is on screen?
      entry.target.classList.add('visible');  // add .visible class
      observer.unobserve(entry.target);   // stop watching it
    }
  });
}, { threshold: 0.1 });   // fire when 10% of element is visible

// Step 3: Start watching every reveal element
revealEls.forEach(el => observer.observe(el));
CSS classStarting position → Animates to
.revealInvisible, 28px below its final position → fades in, slides up to normal position
.reveal-leftInvisible, 28px to the left → fades in, slides right to normal position
.reveal-rightInvisible, 28px to the right → fades in, slides left to normal position
.reveal.visibleopacity: 1, transform: none — the end state added by JavaScript
Why unobserve() after first trigger?
Once an element has animated in, we never need to watch it again. Calling observer.unobserve() removes it from the watch list — less work for the browser, better performance on pages with many animated elements.

Music System

The complete music implementation — why mobile browsers block autoplay, how we worked around it, and how the toggle button works. Every line explained.

Why Autoplay is Blocked (and How We Fix It)

The Browser Rule
Since 2018, all major browsers (Chrome, Safari, Firefox) block audio/video from auto-playing with sound. This was added to stop annoying ads from blasting audio. The browser requires a "user gesture" — a tap, click, or key press — before any audio can play.
Music autoplay strategy — two-path approach
Page loads
→
Try audio.play() immediately
→
Desktop: works ✓
Same play() call
→
Mobile: blocked, promise rejects
→
.catch() silently swallows the error
Mobile fallback
→
Listeners on click/touchstart/scroll
→
First gesture → music starts ✦
The { once: true } option means each listener removes itself after firing — it only triggers once.
JavaScript — Full music system (lines 1300–1330)
/* ── Setup ── */
const audio = document.getElementById('bgAudio'); // find the <audio> tag
const btn   = document.getElementById('music-btn'); // find the toggle button
audio.volume = 0.32;   // set to 32% volume — not too loud
let musicStarted = false; // flag: has music started yet?

/* ── Start function (called once on first gesture) ── */
function startMusic() {
  if (!musicStarted) {             // only run if not already started
    audio.play().then(() => {       // try to play
      musicStarted = true;          // mark as started
      btn.classList.add('playing'); // activate equalizer animation
    }).catch(() => {});             // silently ignore any error
  }
}

/* ── Attach to first user gesture (any of these 4 events) ── */
['click', 'touchstart', 'keydown', 'scroll'].forEach(ev => {
  document.addEventListener(ev, startMusic, {
    once: true,    // auto-removes after firing once
    passive: true  // scroll performance optimisation
  });
});

/* ── Toggle button: pause/resume ── */
btn.addEventListener('click', (e) => {
  e.stopPropagation();  // don't let click bubble up & trigger startMusic again
  if (audio.paused) {
    audio.play().then(() => {
      btn.classList.add('playing');
      musicStarted = true;
    });
  } else {
    audio.pause();
    btn.classList.remove('playing'); // stops equalizer animation
  }
});
Why stopPropagation() on the button?
When you click the music toggle button, the click event travels up the DOM (this is called "event bubbling") — it would also trigger the startMusic listener on document. stopPropagation() stops the event from bubbling, preventing a double-trigger.

JavaScript — submitRSVP() Line by Line

JavaScript — submitRSVP function (lines 1348–1377)
function submitRSVP(e) {
  e.preventDefault();  // stop form from reloading the page (default HTML behaviour)

  // Collect values from the 3 form fields
  const name       = document.getElementById('rsvp-name').value.trim();
  //                 ↑ gets the text input    ↑ removes leading/trailing spaces

  const attendance = document.querySelector('input[name="attendance"]:checked');
  //   Finds whichever radio button (Yes/No/Maybe) is currently selected

  const guests = document.getElementById('rsvp-guests').value || 1;
  //  Number input value, defaults to 1 if left empty (|| 1)

  if (!name || !attendance) return; // stop if required fields are empty

  // Disable button to prevent double-submit
  const btn = document.querySelector('.rsvp-btn');
  btn.textContent = 'Sending…';
  btn.disabled = true;

  // Send data to Google Apps Script — FIRE AND FORGET
  fetch(APPS_SCRIPT_URL, {
    method: 'POST',
    body: JSON.stringify({ name, attendance: attendance.value, guests })
    // JSON.stringify converts JS object → string: '{"name":"Raj","attendance":"yes","guests":"3"}'
  }).catch(() => {}); // if network fails, ignore — don't crash the UI

  // Start fade-out animation on the form
  const form = document.getElementById('rsvp-form');
  const msg  = document.getElementById('rsvp-confirm-msg');
  form.style.opacity = '0';            // make form transparent
  form.style.transition = 'opacity 0.4s'; // over 0.4 seconds

  // After 400ms (when fade is done), hide form and show message
  setTimeout(() => {
    form.style.display = 'none';
    msg.style.display  = 'block';
    if (attendance.value === 'yes') {
      msg.textContent = `Dear ${name}, we are so delighted!…`;
      // Template literal: ${name} inserts the guest's actual name
    } else {
      msg.textContent = `Dear ${name}, we shall miss your presence…`;
    }
  }, 400);  // 400ms delay matches the CSS transition duration
}

Google Apps Script — The Server-Side (Cloud Function)

What is Google Apps Script?
It's a free JavaScript runtime hosted by Google. You write code in their editor, deploy it as a "web app", and it gets a public HTTPS URL. When our website sends a POST request to that URL, Google runs the script instantly in their cloud — no server, no hosting cost, no maintenance.
Google Apps Script — deployed as web app (runs on Google's servers)
// doPost() is called automatically when a POST request arrives
function doPost(e) {
  // Get the active Google Sheet (the one this script is attached to)
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();

  // Parse the JSON string sent by our website back to a JS object
  const data = JSON.parse(e.postData.contents);
  // data is now: { name: "Raj", attendance: "yes", guests: "3" }

  // Add a new row to the spreadsheet with 4 values
  sheet.appendRow([new Date(), data.name, data.attendance, data.guests]);
  //              ↑ current timestamp  ↑ name      ↑ yes/no/maybe  ↑ count

  // Send back a JSON success response to the website
  return ContentService.createTextOutput(JSON.stringify({ status: 'ok' }))
    .setMimeType(ContentService.MimeType.JSON);
}

// doGet() is called when the dashboard page loads data
function doGet(e) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  const rows  = sheet.getDataRange().getValues(); // all rows as 2D array
  const keys  = rows[0].map(k => k.toString().toLowerCase()); // header row

  // Convert rows to objects: [ {timestamp:…, name:…, attendance:…}, … ]
  const result = rows.slice(1).map(r =>
    Object.fromEntries(keys.map((k, i) => [k, r[i]]))
  );
  return ContentService.createTextOutput(JSON.stringify(result))
    .setMimeType(ContentService.MimeType.JSON);
}

Dashboard & Deployment

How the /dataofsanya admin page reads and displays all RSVP data, and how GitHub Pages serves the site to the world for free.

dataofsanya/index.html — RSVP Dashboard

JavaScript — loadData() in dataofsanya/index.html
async function loadData() {
  try {
    // Send GET request to Apps Script — gets all sheet rows as JSON
    const res  = await fetch(APPS_SCRIPT_URL);  // wait for response
    const data = await res.json();              // parse JSON → JS array

    document.getElementById('loading').style.display = 'none'; // hide spinner

    if (!data.length) { /* show "no responses" message */ return; }

    let yes = 0, no = 0, guests = 0;
    const tbody = document.getElementById('rsvp-tbody');

    data.forEach((row, i) => {
      const att = (row.attendance || '').toLowerCase();
      if (att === 'yes') yes++;   // count attending
      if (att === 'no')  no++;    // count not attending
      guests += parseInt(row.guests) || 0; // total guest count

      // Create a table row element dynamically
      const tr = document.createElement('tr');
      const date = new Date(row.timestamp)
        .toLocaleDateString('en-IN', {…}); // format: "15 Mar 2025, 3:42 PM"

      tr.innerHTML = `<td>${i+1}</td><td>${row.name}</td>
         <td><span class="badge badge-${att}">${att}</span></td>
         <td>${row.guests}</td><td>${date}</td>`;
      tbody.appendChild(tr); // add row to the <tbody> in the HTML
    });

    // Update the 4 stat boxes at the top
    document.getElementById('count-total').textContent  = data.length;
    document.getElementById('count-yes').textContent    = yes;
    document.getElementById('count-no').textContent     = no;
    document.getElementById('count-guests').textContent = guests;

    document.getElementById('rsvp-table').style.display = 'table'; // show table

  } catch (err) {
    // If fetch fails (offline, script error), show error message
    document.getElementById('error-msg').style.display = 'block';
  }
}
loadData(); // runs immediately when page opens
