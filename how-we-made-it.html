Technical Documentation — Silver Jubilee Invitation 01

What Is It?

A digital wedding anniversary invitation — opened on any phone or computer, no app needed, works forever. Let's break down every piece like you're five.

🏠
The Website Itself
HTML / CSS / JavaScript
HTML is like the bricks of a house — it puts text, photos, and buttons on screen. CSS is the paint and furniture — it makes everything look beautiful with colors, fonts, and animations. JavaScript is the electricity — it makes things move, play music, and respond when you tap.
📦
Where It Lives
GitHub Pages
Imagine a free locker on the internet that holds your files and shows them to the whole world. GitHub Pages is exactly that — free forever, no monthly fee, no server to manage. Every time we saved a change it went live in under 2 minutes.
🎵
Background Music (Saathiya)
Web Audio API
Phones are sneaky — they block music from auto-playing so ads can't annoy you. We worked around this: the song waits silently, then the moment you tap or scroll anywhere on the page, it starts. An animated equalizer (4 bouncing bars) shows it's playing.
📷
The Couple Photos
CSS object-fit + blur layers
Photos come in all shapes. To make any photo fit a perfect circle without ugly white gaps, we stack two copies — the back copy is the same photo, heavily blurred and zoomed in to fill the gaps. The front copy is sharp and shows the actual subject. Like frosted glass behind a picture frame.
📋
RSVP Form
Google Apps Script + Google Sheets
When a guest fills the form and clicks Confirm, their name, attendance, and guest count is sent to a tiny program running in the cloud (Google Apps Script). That program writes a new row into a Google Spreadsheet — permanently, with a timestamp. The host can view all responses at /dataofsanya.
Technical Documentation — Silver Jubilee Invitation 02

How It Was Built

Step by step — from blank page to 10 live designs. No design software, no templates. Every pixel hand-coded.

1
Structure the page (HTML)
Wrote all the sections: hero, Then & Now, Event Details, Moments, Quote, RSVP, Footer. Each section is a <section> tag — like chapters in a book.
2
Style everything (CSS)
Applied fonts from Google Fonts (Playfair Display, Lato, Noto Devanagari), colors, spacing, animations. CSS variables let us change the whole color scheme by editing 9 lines.
3
Draw the decorations (SVG)
Every botanical border, leaf wreath, and corner accent is a hand-drawn SVG shape in code — not an image file. SVGs scale perfectly to any screen size.
4
Add motion (JavaScript)
IntersectionObserver watches when each section enters the screen and adds a "visible" class — triggering CSS slide-up animations as you scroll.
5
Add music
Linked the MP3 file stored in /audio/. JavaScript tries to play immediately (works on desktop), and also listens for any touch/click/scroll to start on mobile.
6
Deploy to the internet
Used Git to push all files to GitHub. GitHub Pages automatically published the site. A .nojekyll file tells GitHub not to process the files — just serve them as-is.
How scroll reveal animations work
Page loads
→
All sections start invisible (opacity: 0)
→
Viewport watches each section

This uses a built-in browser tool called IntersectionObserver — no extra code library needed.
📱 Mobile-First Design
The invitation works on any screen size. We used CSS clamp() for font sizes — instead of a fixed size like 32px, we wrote clamp(1.6rem, 5vw, 2.4rem) which means "between 1.6 and 2.4, scale based on screen width." Two-column layouts collapse to single column on phones using CSS Grid with a breakpoint at 600px.

Data & Code

The two trickiest parts: making music work on mobile, and saving RSVP data forever. Here's exactly how both work.

🎵 Mobile-Safe Music Autoplay
Browsers block music from auto-playing. Our solution: try to play immediately (works on desktop), and also attach listeners for the first tap, scroll, or keypress on mobile. Once triggered, all listeners remove themselves so it only fires once.
JavaScript — audio/index.html
const audio = document.getElementById('bgAudio');
audio.volume = 0.32;

// Try immediately (works on desktop)
audio.play().catch(() => {});

// Fallback: wait for first user gesture on mobile
const start = () => {
  audio.play().catch(() => {});
  ['click', 'touchstart', 'scroll'].forEach(e =>
    document.removeEventListener(e, start));
};
['click', 'touchstart', 'scroll'].forEach(e =>
  document.addEventListener(e, start, { once: true }));

🔵 The Circular Photo Blur Trick
The Problem
Photos are rectangular. A circle crop creates white/empty gaps on the sides if the photo is portrait-shaped. object-fit:contain shows the full photo but leaves gaps.
The Solution
Stack two layers inside overflow:hidden circle. Bottom layer: same image, blur(40px) scale(1.3). Top layer: sharp photo. The blurred layer fills all gaps with matching tones.
